# Design: Single-Source Skills Installer

**Date:** 2026-06-26
**Status:** Approved

## Problem

The repo currently has skill content in more than one place. Authors can edit or add
a skill under `skills/`, while installed copies may also exist under agent-native
directories such as `.agents/skills/` or `.claude/skills/`. That makes skill updates
easy to miss because generated copies can drift from the intended source.

The bootstrap flow is also not streamlined: users download an executable named
`install`, while the requested interface is an explicit `install.sh` script. The
installer currently supports only Claude Code and Codex through hardcoded branches,
so adding another platform would require editing multiple parts of the script.

## Goals

- Make `skills/<skill-name>/...` the only canonical skill source.
- Treat `.agents/skills/`, `.claude/skills/`, and future agent destinations as
  generated install outputs that point to the shared cache by default.
- Provide an `install.sh` bootstrap entry point while preserving existing `install`
  users.
- Make platform support easy to extend from one centralized definition.
- Keep install behavior idempotent and compatible with the current test suite.

## Non-Goals

- No per-skill selection.
- No per-platform install mode differences; every platform uses the selected
  global mode.
- No Windows-specific support.
- No migration to Python or another runtime.

## Architecture

The repo keeps a single source catalog:

```text
skills/
  <skill-name>/
    SKILL.md
    ...
```

Agent-specific directories are generated by the installer:

```text
.claude/skills/<skill-name>/
.agents/skills/<skill-name>/
```

The generated directories should be ignored by this repo and by projects that run
the installer. Existing committed generated files should be removed from git
tracking during implementation without deleting the canonical `skills/` content.
By default, generated skill entries are symlinks to
`$SKILLS_CACHE/skills/<skill-name>` so both agent paths share one editable source.

## Installer Design

`install.sh` becomes the real installer script. The existing `install` file remains
as a compatibility wrapper that executes `install.sh` from the same directory.

The installer keeps the existing cache behavior:

- `SKILLS_REPO` defaults to the GitHub skills repo.
- `SKILLS_CACHE` defaults to `~/.local/share/skills`.
- First run clones the cache.
- Later runs fetch and reset to the upstream branch.
- If the cache has local changes, the installer warns, skips fetch/reset, and
  installs from the dirty cache.
- If fetch fails and a cache exists, the installer warns and uses the cached copy.

The installer then installs every valid skill directory from
`$SKILLS_CACHE/skills/*/` into the selected platform destinations. A valid skill
directory must contain `SKILL.md`; invalid directories are skipped with a warning.
The default install mode is `symlink`. `SKILLS_INSTALL_MODE=copy` installs real
copied directories instead, and symlink failures warn and fall back to copying
that skill.

## Platform Registry

Supported platforms are defined once in the script through a small registry with
three fields:

```text
name:destination:exclude-entry
```

Initial entries:

```text
claude:.claude/skills:.claude/
codex:.agents/skills:.agents/
```

Argument handling is derived from this registry:

- No argument means `all`.
- `all` installs every registered platform.
- A platform name installs only that platform.
- Unknown names exit with an error that lists supported platform names.

Adding a platform should require one new registry row and matching tests. The install
loop, directory creation, and git exclude handling should not need platform-specific
branches.

## Generated Output Policy

The repo should include a committed `.gitignore` entry for generated agent
directories:

```text
.claude/
.agents/
```

The installer should continue to add the selected generated directories to the
target project's `.git/info/exclude`, preserving the current behavior of not
editing the target project's committed `.gitignore`.

## Testing

The test suite should continue to use temporary git repos and local cache fixtures.
It should cover:

- Running outside a git repo exits with an error.
- Missing cache plus unreachable remote exits non-zero.
- Local-only cache installs successfully.
- Unknown platform exits with supported platform names.
- `claude` installs only `.claude/skills`.
- `codex` installs only `.agents/skills`.
- Default `all` installs both registered platforms.
- Default install mode symlinks each skill entry to the shared cache.
- `SKILLS_INSTALL_MODE=copy` installs real directories.
- Skill directories without `SKILL.md` are skipped with a warning.
- `.git/info/exclude` entries are added only for selected platforms.
- Exclude entries are idempotent.
- The target project's `.gitignore` is untouched.
- A full run against the real skills repo is idempotent.

Tests should assert behavior through the platform registry rather than duplicating
hardcoded script structure where possible.

## Documentation

`README.md` should be updated to:

- Use `install.sh` in the bootstrap command.
- Explain that `skills/` is the single source of truth.
- Explain that `.claude/` and `.agents/` are generated outputs linked to the
  shared cache by default.
- Document the one-row process for adding another supported platform.

## Rollout

Implementation should happen in a small sequence:

1. Add `install.sh` and keep `install` as a compatibility wrapper.
2. Refactor platform handling into the registry.
3. Update tests for the registry-based behavior.
4. Add repo-level ignores for generated agent directories.
5. Remove tracked generated skill copies from `.claude/skills`.
6. Update README documentation.
7. Run the installer test suite.
